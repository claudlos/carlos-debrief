### 🦞 Lobste.rs

**[It's NOT OK to compare floating-points using epsilons](https://lisyarus.github.io/blog/posts/its-ok-to-compare-floating-points-for-equality.html)** [1]
*Lobste.rs • compsci • 2026-04-14*
This essay argues that blanket epsilon-based floating-point comparisons are often the wrong fix, and that programmers should reason from the numerical structure of each problem instead of applying one tolerance everywhere. It matters because graphics, simulation, and scientific code still ship subtle correctness bugs when developers treat floating-point error as a generic afterthought.

*Lobste.rs • security • 2026-04-14*
This historical chapter revisits Intel's iAPX 432, a capability-oriented architecture designed around protected objects and fine-grained authority control rather than flat ambient access. It matters because modern supply-chain security and sandboxing debates keep rediscovering capability-security ideas that hardware experiments like the 432 explored decades ago.

*Lobste.rs • security • 2026-04-14*
This announcement surfaces a new KeePassXC fork hosted on Codeberg, signaling continued experimentation and governance branching in open-source password manager tooling. It matters because security-sensitive utilities live or die by maintenance trust, so even small ecosystem forks can reshape where users place their credentials and updates.

### 🤗 HuggingFace

**[Time is Not a Label: Continuous Phase Rotation for Temporal Knowledge Graphs and Agentic Memory](https://huggingface.co/papers/2604.11544)** [14]
*Weixian Waylon Li, Jiaxin Zhang, Xianan Jim Yang, Tiejun Ma, Yiwen Guo*
RoMem models time in agent memory with continuous phase rotation and a learned volatility score so persistent facts stay stable while outdated facts are rotated out of phase instead of being deleted. It matters because long-lived agents need memory systems that can distinguish durable knowledge from fast-changing state without constant LLM-mediated cleanup.

**[Audio-Omni: Extending Multi-modal Understanding to Versatile Audio Generation and Editing](https://huggingface.co/papers/2604.10708)** [15]
*Zeyue Tian, Binxin Yang, Zhaoyang Liu, Jiexuan Zhang, Ruibin Yuan*
Audio-Omni proposes an end-to-end framework that unifies audio understanding, generation, and editing across speech, music, and general sound, backed by a million-pair AudioEdit dataset. It matters because audio AI is still fragmented across narrow expert systems, and a single multimodal stack could make general-purpose audio agents much more practical.

**[IceCache: Memory-efficient KV-cache Management for Long-Sequence LLMs](https://huggingface.co/papers/2604.10539)** [16]
*Yuzhen Mao, Qitong Wang, Martin Ester, Ke Li*
IceCache clusters semantically related tokens and combines that layout with PagedAttention to keep long-context LLM inference accurate while moving more cache data off GPU. It matters because KV-cache memory is one of the main bottlenecks for long-context serving, especially on constrained hardware and agent workloads with long traces.

**[ATANT: An Evaluation Framework for AI Continuity](https://huggingface.co/papers/2604.06710)** [17]
*Samuel Sameer Tanguturi*
ATANT defines continuity as a measurable systems property and introduces a 250-story benchmark for testing whether AI memory stacks can preserve, update, and retrieve narrative truth over time without cross-contamination. It matters because many agent products claim memory, but few have rigorous evaluations showing that memory remains correct once many narratives coexist.

**[TRACE: Capability-Targeted Agentic Training](https://huggingface.co/papers/2604.05336)** [18]
*Hangoo Kang, Tarun Suresh, Jon Saad-Falcon, Azalia Mirhoseini*
TRACE turns failed agent trajectories into capability-targeted synthetic RL environments, then trains specialized LoRA adapters and routes the agent to the right one at inference time. It matters because agent improvement is starting to move from generic post-training toward explicit repair of the concrete capabilities that caused failure.

**[Efficient RL Training for LLMs with Experience Replay](https://huggingface.co/papers/2604.08706)** [19]
*Charles Arnal, Vivien Cabannes, Taco Cohen, Julia Kempe, Remi Munos*
This study argues that strict on-policy sampling is often wasteful for LLM post-training and shows that well-designed replay buffers can reuse experience without sacrificing final performance. It matters because rollout generation is expensive, so any credible path to lower inference cost changes the economics of reasoning and agent training.

**[Agentic Aggregation for Parallel Scaling of Long-Horizon Agentic Tasks](https://huggingface.co/papers/2604.11753)** [20]
*Yoonsang Lee, Howard Yen, Xi Ye, Danqi Chen*
AggAgent treats parallel agent trajectories as an environment of their own and gives an aggregation agent tools to inspect, compare, and synthesize them rather than simply voting on final answers. It matters because deep-research style systems increasingly rely on many long rollouts, and smarter aggregation is becoming a first-class scaling lever.

**[Advancing Polish Language Modeling through Tokenizer Optimization in the Bielik v3 7B and 11B Series](https://huggingface.co/papers/2604.10799)** [21]
*Krzysztof Ociepa, Łukasz Flis, Remigiusz Kinas, Krzysztof Wróbel, Adrian Gwoździej*
This report shows how a Polish-specific tokenizer and adapted pretraining curriculum improve efficiency and usable context length for Bielik v3 compared with more universal vocabularies. It matters because multilingual models still underperform on morphologically rich languages when tokenization is optimized for broad coverage instead of local linguistic structure.

**[Not All Denoising Steps Are Equal: Model Scheduling for Faster Masked Diffusion Language Models](https://huggingface.co/papers/2604.02340)** [22]
*Ivan Sedykh, Nikita Sorokin, Valentin Malykh*
This paper shows masked diffusion language models can swap in smaller models for less sensitive denoising steps, cutting sampling FLOPs while preserving much of the quality. It matters because diffusion LMs remain computationally expensive, and step-aware scheduling offers a simple route to cheaper generation.
